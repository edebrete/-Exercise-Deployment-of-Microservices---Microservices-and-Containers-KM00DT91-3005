Auction System Project based on microservices and DDD
Project execution flow: SellerService -> ProductForSaleEvent -> BidderService -> BidEvent -> AuctionManagerService.
The system starts with 5 bidders. Each bidder sends at most 1 bid per product.
Tech-stack chosen for this project: 
Spring Boot (Java), databases in Postgresql, event-driven-comunication with Kafka, conteinarization with Docker, configuration files in YAML.

What needs to be installed to run this project on Windows 10+: Docker Desktop, WSL2.

To run this project on Windows, start Docker Desktop, then open a command prompt in this folder and run the following command:
powershell -ExecutionPolicy Bypass -File run.ps1

To test the functionality, run the following type of commands:
curl -X POST "http://localhost:8080/sell?desc=DemoProduct1"
curl -X POST "http://localhost:8080/sell?desc=DemoProduct2"

Expected output in cmd:
Published

Notice: Wait for a minute after starting the program before running commands;
if at first response is "curl: (52) Empty reply from server", wait some time and try again.
View Logs in Docker Desktop app. The winner is announced by AuctionManager in its logs 10 seconds after the first bid on each product.

To stop the project and clear everything generated by it, run the following command:
docker compose down -v
